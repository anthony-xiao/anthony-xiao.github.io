Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?

An analogy I can think of that can describe HTML CSS, and the DOM would be comparing this to buildig a house. The HTML is the structure and frame work of the house, it is what everything will go off of from. CSS is what is being built on top of the structure (HTML) to make it more appealing. The DOMS is like the carpenter that has been handed the structure and design, if you follow it it will be built as designed but each builder are different and have their own intepretation on things, this can at times cause slight differences between builders (DOM) while given the same information.

What is meant by boxifying design?

Boxifying desing means to look at webpages as a combination of boxes. Everything you see on a webpage is something within a box, and sometimes a box within a box. Making everything in a box allows things on a webpage to be designed and moved easier. Even with items that are circle or other odd shapes, it begins with a box and further edited/coded into its shape.

What is the box model?

The box model is the breakdown of how 'box' look like during coding. It has 4 components

  • Content - Area that the actual content (picture, text, etc) is taking up.
  • Padding - This is gives an area around the content that is still within the box
  • Border - Borders incapsulate content and padding, it will have the color property of the box so it can be as the title suggests a border or invisible if the color is transparent.
  • Margin - This is area that goes outside of the box, it gives space between the box and the next item that will be next to the box.
